Bonus Work — No Extra Charge

All of these were outside the original migration scope but were caught and fixed during the pre-handoff audit and shipped at no charge. Everything below is already live in production.

Frontend & UX
  • Mobile navigation built from scratch Below 768px the desktop nav was hidden with no replacement — mobile users couldn't reach A-Z, Shop, Library, or any other section. New MobileNav.tsx with Radix Sheet drawer, hamburger trigger, full keyboard / focus-trap accessibility, brand-styled with proper contrast.
  • Apple Pay + Google Pay enabled at checkout A site-wide Permissions-Policy header was silently blocking the wallet buttons in checkout. Fixed by allowlisting Stripe + Google Pay origins. iPhone users now see Apple Pay; Chrome users see Google Pay.
  • Banner contrast fixes site-wide (WCAG AA) 4 components were using same-hue dark text on light tinted backgrounds (text-rose-700 on bg-rose-50, etc.) — broken in macOS dark mode and below WCAG AA. Switched to saturated bg + white text pattern: FeatureTicker rotating banner, Home shop+stats banners, EmailVerificationBanner, ImportRegulationNotice.
  • Out-of-stock vendor row UI redesign "Where to Buy" rows were using disabled:opacity-60 on the entire row, making available vendors look broken too. Now greyed bg + "Out of stock" pill + clear "Unavailable" disabled-styled button. Available vendors stand out properly.
  • Footer accessibility upgrade Link contrast was 4.2:1 (below WCAG AA's 4.5:1 minimum). Bumped to compliant ratio. Also caught and replaced 18 stale support@rosebase.app references — all now point to verified support@rosesroses.com.
  • Privacy/Terms/Refunds legal placeholder filled All three pages contained literal [Legal Entity Name, LLC] placeholder text where the company name should be — legally significant on PII-collecting pages. Filled with RosesRoses (Fritz Companies).
Backend & Security
  • Entire transactional email pipeline repaired 8 hardcoded sender addresses in production code were using unverified domains (rosebase.com / rosebase.app), causing every customer order confirmation, nursery notification, admin alert, shipped notification, and audit report email to silently 403 from Resend. Plus 2 module-level fallback defaults and 1 broken template literal. Fixed at every callsite.
  • Admin auth hardening Admin password was defaulting to literal "admin123" if the env var was missing — fail-OPEN authentication. Removed default; logs error and skips admin seeding if missing. Generated cryptographically strong random password, stored in AWS Secrets Manager.
  • Stripe live publishable key swap Frontend was using a Stripe TEST publishable key in production. Customers literally could not pay — the Stripe Payment Element returned 400. Fixed by swapping to pk_live_* in the Dockerfile build args.
  • Checkout NaN price validation Two endpoints (/checkout/update-intent and /checkout/confirm) accepted unvalidated price math — NaN values could propagate through tax calculation and reach Stripe. Added Number.isFinite guards at the entry points.
  • Stripe webhook safety net New POST /api/webhook/stripe handler with signature verification. If a customer's browser disconnects between Stripe success and our /checkout/confirm call, the webhook detects the stuck order and emails the admin with full details for manual reconciliation. Was technically in scope (Phase 7) but built well beyond the minimum.
  • OpenAI silent-failure fix across 24 enrichment scripts Models had been upgraded to gpt-5-mini / gpt-5.2 mid-migration. These models REJECT the temperature parameter, causing every affected enrichment call to silently drop the record. Surgical removal of temperature: at 24 callsites across 18 files. Verified zero remaining references.
Operations & Reliability
  • Auto-retire stale nursery scrapers (entirely new feature) A new weekly job (Sundays 8am ET) automatically retires nurseries with no successful catalog scrape in 60+ days. Conservative thresholds (60 days, 20+ attempts), capped at 50 retirements per run, full audit trail in scraper_triage, sends admin email summary. Without this, the "failed scrapers" count grew unbounded forever — now it stays bounded with zero ongoing effort.
  • Operations manual expanded 4x Original draft documented 6 of 24+ scheduled jobs. Now covers every nightly, weekly, monthly, quarterly, annual, and admin-triggered worker with mission, schedule, and runbook. Plus the schedule contradiction in weekly-event-discovery is flagged inline.
  • CloudWatch alarm tuning The rosebase-app-down alarm was firing on 2 consecutive 60-second failures — too sensitive for normal task replacements during nightly cron. Retuned to 3-of-5-minute evaluation. Real outages still page within ~3 minutes; transient blips no longer wake anyone up.
  • Admin dashboard false-positive alert eliminated The "X nightly job(s) overdue: dead-url-check" alert had been firing constantly because the dashboard's overdue-filter wasn't reading the triggerMode field added in PR #11. Imported the registry, added the filter logic. Admin-triggered workers no longer false-flag.
  • Migration drift CI guard New CI job that fails any PR adding a duplicate-prefix migration file (uses git diff against base). Pre-existing duplicates pass; future ones can't slip in. Documented in MIGRATION-CLEANUP-NOTES.md.
  • RDS Performance Insights enabled 7-day retention free tier, no restart, no downtime. Read-only query performance monitoring. Useful for diagnosing slow queries during cron windows or as traffic grows.
  • S3 BlockPublicPolicy hardened Public bucket had BlockPublicPolicy=false — a foot-gun where someone could accidentally make objects publicly readable. All 4 public-access flags now ON. Closes the misconfiguration risk without breaking any current functionality (CloudFront OAC bypasses these blocks).
